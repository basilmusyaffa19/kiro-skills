# Example Rules

## Example Neutrality

- Examples MUST use generic placeholders.
- Examples MUST NOT reference real data from source documents.
- Examples MUST NOT contain specific entities, values, or domain-specific names.
- Examples MUST represent the transformation pattern only.
- Examples MUST remain domain-neutral whenever possible.

## Structural Examples Only

- Examples MUST demonstrate logic using abstract placeholders.
- Examples MUST NOT mirror real document structure.
- Examples MUST describe relationships such as: parent-child hierarchy, grouping, aggregation, transformation rules.

## Avoid Example Overfitting

- Examples MUST NOT repeat tokens present in the source input.
- Examples MUST NOT include domain-specific vocabulary unless strictly required.
- Examples MUST remain minimal and abstract.
